Автоматическое динамическое масштабирование требует редизайна алгоритма.
Масштаб может изменяться по ходу построения.
Silent Mode - для низкопроизводительных систем, визуализация только по завершении
получения всех результатов.

File   : dialog_memory_draw.inc.
Branch : WM_PAINT.

Notes:

1) Необходимость ревизуализации и координаты обновляемых регионов определять в процедуре
   WM_TIMER, а в процедуре WM_PAINT дополнительно проверять.

2) Параметр N = количество измерений и точек графика, протяженность оси X.
   Предопределенный массив:
   size[N] = размеры измерительных блоков,
   Массив результатов измерений:
   cpi[N] = clocks per instruction,  operation time = K * cpi,  
   K = clock period, ns.
   Массив, вычисленный на основании результатов измерений:
   mbps[N] = megabytes per second, operation speed = 1 / operation time.
   nspi[N] = nanoseconds per instruction, operation latency = operation time.

3) Параметр i = текущая измерительная итерация, считая от 0.

Steps:

1)  Построение координатных осей X,Y. Собственно оси не зависят от результатов измерений
    и динамического автоматического масштабирования, изменяются только подписи числовых 
    значений для оси Y. Поэтому построение осей можно выполнить только при старте, на
    первом проходе, один раз.
    Графическая операция выполняется в транзитном буфере.

2)  Построение подписей для оси X, на первом проходе, один раз.
    Графическая операция выполняется в транзитном буфере.

3)  Построение дампа, на первом проходе, один раз.
    Графическая операция выполняется в транзитном буфере.

4)  Получение информации о результатах измерений, массивы cpi[N], nspi[N], mbps[N].
    Определение необходимости обновления для графика функции и таблицы статистики.

5)  Вычисление статистики,
    величины: min, max, median, average
    для параметров: cpi, nspi, mbps.

6)  Определение необходимости изменения масштаба, на основании величины Ymax.
    Построение подписей для оси Y, в соответствии с текущим используемым масштабом.
    До получения первого результата измерения устанавливается значение по умолчанию
    default Ymax.
    Графическая операция выполняется в транзитном буфере.

7)  Построение графика Y=F(X), ломаная линия. В зависимости от измерительного
    сценария, используется информация из массивов:
    пропускная способность, mbps[N], megabytes per second,
    латентность, nspi[N], nanoseconds per instruction.
    Графическая операция выполняется в транзитном буфере.

8)  Визуализация, передача информации из транзитного буфера в видео адаптер.
    Функции BeginPaint-BitBlt-EndPaint.
